Commande courante de

  • cd +_nom_du_dossier/ // pour se rendre dans un dossier ! => le "/" n'est pas obligatoire
  • cd .. // pour remonter d'un cran dans l'arborescence
  • cd ~ // pour revenir dans le dossier principal, ~ = alt + n pour
  • pwd // pour afficher le fil d'Ariane
  • ls // pour voir la liste des fichiers et dossiers dans le dossier courant
  • mkdir +_nom_du_dossier // pour créer un nouveau dossier
  • touch +_nom_du_fichier // pour créer un nouveau fichier
  • cat +_nom_du_fichier // permet d'afficher le contenu d'un fichier
  • git config --global user.name "+_votre_nom" // pour définir le nom utilisé pour github.com
  • git config --global user.email "+_votre_email" // pour définir l'email utilisé pour github.com
  • git config user.name // permet de voir le nom utilisé pour github.com
  • git config user.email // permet de voir l'email utilisé pour github.com
  • git init // permet d'initialiser le répertoire courant
  • git status // permet de savoir dans quel état le projet git se trouve
  • git add +_nom_du_dossier_ou_nom_du_fichier // permet d'ajouter à l'index les fichiers et dossiers ainsi que le contenu des dossiers
  • git add -A // permet d'ajouter tous les fichiers et dossiers d'un coup
  • git commit -m "+_le_message" // permet de commiter et d'ajouter un message au commit
  • git commit -am " +_le_message" // permet de mettre à jour les fichiers déjà existants dans son index
  • git log // permet d'afficher l'historique des commits, q pour quitter log
  • git lg // raccourci de git log pour afficher l'historique des commits sous forme de branche
  • git diff // permet d'afficher les différences dans la branche
  • git checkout SHA_du_commit // permet de revenir au commit du SHA sélectionner
  • git checkout master // permet de revenir à la branche principale au commit le plus récent
  • git clone +_url_du_clone // permet de récupéré le clone du projet présent sur github
  • git push origin master // permet d'envoyer les modifications dans la branche master du remote origin
  • git pull origin master // permet de récupéré uniquement les fichiers modifiés
  • git branch // affiche la ou les branches du projet et permet de voir sur quelle branche on se trouve grâce à la petite étoile
  • git branch +_nom_de_la_nouvelle_branche // permet de créer une nouvelle branche dans le projet
  • git branch -d +_nom_de_la_branche // permet de supprimer la branche appeler
  • git branch -D +_nom_de_la_branche // permet de supprimer la branche appeler même si elle n'a pas été commiter merger
  • git checkout +_nom_de_la_branche // permet de rentrer dans la branche voulue
  • git checkout -b +nom_de_la_branche // permet de créer ET de rentrer dans la branche
  • git merge +_nom_de_la_branche // permet de fusionner les branches
  • git blame +_nom_du_fichier // permet de voir quels sont les auteurs de chaque ligne d'un fichier dans un projet
  • git show +_SHA_du_commit // permet de voir les différence dans le fichier
  • .gitignore // permet de créer une liste afin d'ignorer des fichiers
  • git stash // permet de sauvegarder temporairement en mémoire un fichier
  • git stash pop // permet de revenir sur le fichier sauvegarder temporairement
  • git stash apply // permet de garder les modifications du stash
  • SHIFT + Insert // sur windows permet de coller, cmd ⌘ + v pour

Editeur VIM

  • vim +_nom_du_fichier // permet de rentrer dans le ficher et de l'éditer sans passer par un IDE
  • [ECHAP] suivi de :w // permet de sauvegarder
  • [ECHAP] suivi de :q // permet de quitter vim
  • [ECHAP] suivi de :x // permet de sauvegarder et de quitter vim en même temps

Scénario d'un projet GIT

Création et envoi d'un projet vers Github

  • Création en amont d'un repository sur github.com
  • git init
  • git commit -m "+_le_message"
  • git remote add origin https://github.com/+_nom_du_profile/+_nom_du_repository.git
  • git push -u origin master
  • Depuis un projet existant
  • git remote add origin https://github.com/+_nom_du_profile/+_nom_du_repository.git
  • git push -u origin master

Clone d'un projet

  • git clone +_url_du_clone

Mise à jour d'un projet depuis github

  • cd +_chemin_au_dossier
  • git pull origin master

Mise à jour d'un projet vers github

  • cd +_chemin_au_dossier
  • git commit -am " +_message_du_commit_le_plus_détailler"
  • git push origin master
