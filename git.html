Commande courante de dans le terminal

Commande de base terminal

  • cd nom_du_dossier // pour se rendre dans un dossier
  • cd .. // pour remonter d'un cran dans l'arborescence
  • cd ~ // pour revenir dans le dossier principal,~ = alt + n pour
  • pwd // pour afficher le fil d'Ariane
  • ls // pour voir la liste des fichiers et dossiers dans le dossier courant
  • mkdir nom_du_dossier // pour créer un nouveau dossier
  • touch nom_du_fichier // pour créer un nouveau fichier
  • cat nom_du_fichier // permet d'afficher le contenu d'un fichier
  • open nom_du_fichier // permet d'ouvrir le fichier
  • SHIFT + Insert // sur windows permet de coller, cmd ⌘ + v pour
  • ⌘+K clear // permet de clear le terminal

Editeur VIM

  • vim nom_du_fichier // permet de rentrer dans le ficher et de l'éditer sans passer par un IDE
  • [ECHAP] suivi de :w // permet de sauvegarder
  • [ECHAP] suivi de :q // permet de quitter vim
  • [ECHAP] suivi de :x // permet de sauvegarder et de quitter vim en même temps

Commande de base git

  • git config --global user.name "votre_nom" // pour définir le nom utilisé pour github.com
  • git config --global user.email "votre_email" // pour définir l'email utilisé pour github.com
  • git config user.name // permet de voir le nom utilisé pour github.com
  • git config user.email // permet de voir l'email utilisé pour github.com
  • git init // permet d'initialiser le répertoire courant
  • git status // permet de savoir dans quel état le projet git se trouve
  • git add nom_du_dossier_ou_nom_du_fichier // permet d'ajouter à l'index les fichiers et dossiers ainsi que le contenu des dossiers
  • git add . // permet d'ajouter tous les fichiers et dossiers d'un coup
  • git commit --message "le_message" // permet de commiter et d'ajouter un message au commit
  • git commit -m "le_message" // raccourci de --message
  • git commit -am " le_message" // permet de mettre à jour les fichiers déjà existants dans son index
  • git log // permet d'afficher l'historique des commits, q pour quitter log
  • git lg // raccourci de git log pour afficher l'historique des commits sous forme de branche (! ne fonctionne pas sur tous les terminaux)
  • gitk // comme git lg (fonctionne sur tous les OS)
  • git diff // permet d'afficher les différences dans la branche en cours
  • git diff branch_name..branch_name // permet d'afficher les différences entre deux branches
  • git branch // affiche la ou les branches du projet et permet de voir sur quelle branche on se trouve grâce à la petite étoile
  • git branch nom_de_la_nouvelle_branche // permet de créer une nouvelle branche dans le projet
  • git branch -d nom_de_la_branche // permet de supprimer la branche appeler
  • git branch -D nom_de_la_branche // permet de supprimer la branche appeler même si elle n'a pas été commiter merger
  • git checkout nom_de_la_branche // permet de ce placer sur la branche au commit le plus récent
  • git checkout -b nom_de_la_branche // permet de créer ET de rentrer dans la branche
  • git checkout SHA_du_commit // permet de revenir au commit du SHA sélectionner
  • git stash // permet de sauvegarder temporairement en mémoire un fichier
  • git stash pop // permet de revenir sur le fichier sauvegarder temporairement
  • git stash apply // permet de garder les modifications du stash
  • git merge nom_de_la_branche // permet de fusionner les branches
  • git merge --abort nom_de_la_branche // permet d'avorter le merge précédent
  • .gitignore // permet de créer une liste afin d'ignorer des fichiers
  • git reset --hard SHA_du_commit // permet de supprimer le merge contenant le SHA du commit
  • git blame nom_du_fichier // permet de voir quels sont les auteurs de chaque ligne d'un fichier dans un projet
  • git show SHA_du_commit // permet de voir les différence dans le fichier
  • git clone url_du_clone // permet de récupéré le clone du projet présent sur github
  • git remote add origin https://github.com/nom_du_profile/nom_du_repository.git // permet d'ajouté le repository local au repository dans le cloud
  • git push -u origin nom_de_la_branche // permet d'envoyer les modifications de la branche sur le remote origin
  • git pull origin master // permet de récupéré uniquement les fichiers modifiés
